## 🚀 ИНСТРУКЦИИ ДЛЯ ЗАПУСКА

### **1. Backend**

```bash
cd backend

# Установить новую зависимость
npm install express-rate-limit

# Обновить .env файл
# Убедитесь, что JWT секреты минимум 32 символа!
```

**Проверьте `.env`:**
```env
JWT_ACCESS_SECRET=your_very_long_secret_key_at_least_32_characters_here
JWT_REFRESH_SECRET=another_very_long_secret_key_at_least_32_characters
```

**Генерация безопасных секретов:**
```bash
node -e "console.log(require('crypto').randomBytes(32).toString('hex'))"
```

### **2. Telegram Bot**

```bash
cd telegram-bot

# Добавить ключ шифрования в .env
SESSION_ENCRYPTION_KEY=your_64_character_hex_key
```

**Генерация ключа шифрования:**
```bash
node -e "console.log(require('crypto').randomBytes(32).toString('hex'))"
```

**⚠️ ВАЖНО:** Скопируйте сгенерированный ключ в `.env` файл!

### **3. Database**

Если используете production с самоподписанным SSL сертификатом:

1. Скачайте сертификат CA
2. Добавьте в `backend/config/database.js`:
```javascript
ssl: {
    rejectUnauthorized: true,
    ca: fs.readFileSync('/path/to/root.crt').toString()
}
```
